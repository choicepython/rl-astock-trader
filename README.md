# RL-AStock-Trader

基于强化学习(PPO)+9颗子弹动态管理的A股量化交易策略框架。

---

## 🚀 快速开始

### 1. 环境准备

```bash
pip install stable-baselines3 akshare pandas numpy gymnasium
```

### 2. 训练模型

```python
from train_rl_strategy import train_rl_model

# 为指定股票训练专属模型
train_rl_model("002156", timesteps=200000)
```

或直接运行：
```bash
python train_rl_strategy.py
```

### 3. 策略回测

```python
from indicator_strategy import IndicatorStrategy

# 创建策略实例
strategy = IndicatorStrategy(symbol="002156", period="daily")

# 使用训练好的模型进行回测
strategy.backtest_rl("ppo_trading_002156")
```

## 🎯 适合的交易策略类型

### 1. 趋势跟随策略
- **适用场景**：明确的上涨或下跌趋势
- **核心逻辑**：MA5>MA10>MA20多头排列时加仓
- **止盈方式**：追踪止损保护利润

### 2. 波段交易策略
- **适用场景**：震荡上行行情
- **核心逻辑**：分批建仓，快进快出
- **持仓周期**：1-15天

### 3. 动态仓位管理
- **适用场景**：不确定行情方向
- **核心逻辑**：9颗子弹动态加减仓
- **风险控制**：单颗10%仓位，最大90%仓位

## 📚 策略理论依据

### 1. 强化学习框架

本策略基于 **PPO (Proximal Policy Optimization)** 算法，通过与环境交互学习最优交易策略：

```
状态(State) → 动作(Action) → 奖励(Reward) → 策略更新
```

### 2. 9颗子弹仓位管理

将资金分为9颗"子弹"，每颗代表10%仓位：

| 子弹数 | 仓位比例 | 适用场景 |
|--------|---------|---------|
| 1-3颗 | 10-30% | 试探性建仓 |
| 4-6颗 | 40-60% | 确认趋势后加仓 |
| 7-9颗 | 70-90% | 强趋势满仓 |

### 3. 动态止损机制

**双重止损保护**：
- **固定止损**：浮亏超过5%立即止损
- **追踪止损**：盈利超过3%后启动，动态保护利润

### 4. 趋势确认信号

**多头排列奖励**：
- MA5 > MA10 > MA20：额外奖励
- 价格 > MA20：趋势对齐奖励
- 放量突破：量能确认

## 💰 评价体系设计

### 持仓期每日评分

| 评分项 | 规则 | 分值 | 设计目的 |
|--------|------|------|---------|
| **仓位惩罚** | 满仓(90%) | -1.5 | 控制风险，预留子弹 |
| | 80-90% | -0.3 | 轻度提醒 |
| **持仓成本** | 每持有1天 | -0.005/颗 | 鼓励快进快出 |
| **趋势奖励** | 价格>MA20 | +0.2 | 趋势跟随 |
| | 多头排列 | +0.15 | 确认趋势强度 |
| **利润奖励** | 浮盈增长 | ×3.0 | 让利润奔跑 |
| | 浮亏减少 | ×0.3 | 鼓励止损 |
| **大涨奖励** | 2-5% | +(涨幅-2)×2.5 | 捕捉趋势 |
| | 5-8% | +4.0 | 强化持仓 |
| | 8%+ | +10.0 | 超级奖励 |
| **回撤惩罚** | 回撤>5% | 回撤^1.2×0.15 | 非线性惩罚 |
| **持仓天数** | 1-4天 | +0.5 | 最佳周期 |
| | 5-8天 | +0.25 | 理想周期 |
| | 9-12天 | +0.1 | 正常周期 |
| | 13-15天 | -0.15/天 | 惩罚过长 |

### 动作执行评分

| 动作 | 条件 | 奖励 | 设计目的 |
|------|------|------|---------|
| **加仓** | 首次建仓≤7颗 | +0.1/颗 | 鼓励积极建仓 |
| | 逆势加仓 | -0.3/颗 | 防止追高 |
| **减仓** | 盈利≥2% | +6 | 鼓励止盈 |
| | 盈利1-2% | +3 | 小额止盈 |
| | 亏损≥3% | -3 | 及时止损 |
| **全清** | 盈利≥5% | +10 | 大幅止盈 |
| | 盈利3-5% | +6 | 中等止盈 |
| | 盈利1-3% | +3 | 小幅止盈 |

### 清仓评分

| 清仓类型 | 盈亏 | 奖励/惩罚 |
|---------|------|---------|
| **超时清仓** | 亏损>6% | -30 |
| | 亏损4-6% | -8 |
| | 亏损2-4% | -3 |
| | 亏损<2% | -1 |
| | 盈利>8% | +10 |
| | 盈利4-8% | +6 |
| | 盈利2-4% | +3 |
| **止损清仓** | 触发5%止损 | -15 |
| **追踪止损** | 触发追踪 | -10 |

## 📊 观测空间 (22维特征)

### 技术指标 (14维)

| 指标 | 说明 | 用途 |
|------|------|------|
| dif, dea, macd | MACD指标 | 趋势判断 |
| k, d | KD随机指标 | 超买超卖 |
| rsi | 相对强弱指数 | 动量判断 |
| MA5, MA10, MA20, MA60 | 均线系统 | 趋势方向 |
| vol_ratio | 量比 | 量能确认 |
| boll_width | 布林带宽 | 波动率 |
| atr | 真实波幅 | 止损参考 |
| price_ma20_ratio | 价格相对均线比 | 位置判断 |

### 仓位状态 (8维)

| 特征 | 说明 | 用途 |
|------|------|------|
| position_count_norm | 归一化仓位 | 仓位管理 |
| floating_pnl_pct | 浮盈浮亏率 | 止盈止损 |
| hold_days_norm | 持仓天数比例 | 时间管理 |
| avg_cost_price_ratio | 成本价差率 | 盈亏判断 |
| max_drawdown_since_entry | 入场后回撤 | 风险控制 |
| profit_run_days | 连续盈利天数 | 趋势确认 |
| trailing_stop_distance | 追踪止损距离 | 利润保护 |
| atr_multiplier | ATR倍数 | 波动率参考 |
